## How to verify locally

```bash
# 1. Fresh checkout
git clone https://github.com/einperegrin/certpulse.git
cd certpulse
git checkout v0.3.0

# 2. Install + migrate + boot
cp .env.example .env
docker compose up -d

# 3. Health
curl -s http://localhost:3000/health/live | jq        # { status: "ok", ts: "..." }
curl -s http://localhost:3000/health/ready | jq       # { status, timestamp, checks: { db, last_check_age_seconds, last_alert_age_seconds } }

# 4. Metrics (Prometheus exposition format)
curl -s http://localhost:3000/metrics | head -20      # HELP/TYPE lines for certpulse_*

# 5. Rate limiting (after 100 reqs in 60s on /api/*)
for i in $(seq 1 105); do curl -s -o /dev/null -w "%{http_code}\n" -H "Authorization: Bearer $TOKEN" http://localhost:3000/api/domains; done | sort | uniq -c
# → expect mostly 200, then 429 with Retry-After

# 6. Audit log
TOKEN=$(docker compose exec api npx tsx src/cli/tokens.ts create --label verify-bot)
curl -s -H "Authorization: Bearer $TOKEN" "http://localhost:3000/api/audit-log?actor_type=api_token" | jq
# → { rows: [...], total, limit, offset }
```

Open the web UI at `http://localhost:8080/audit` (or whatever port nginx is mapped to) — paginated, filterable by action / actor / resource / date range.

## Notable design choices

- **Rate-limit mounted BEFORE auth** (not after, as the very first commit had it). Comment in `index.ts:147` documents why: a flood of unauthenticated requests can otherwise burn DB CPU on `api_tokens` lookups.
- **`/health/ready` returns 503 when DB is down**, 200 otherwise. This is what k8s liveness/readiness probes and most load balancers expect.
- **Audit `actorId`** is the token label (not the token hash) — the label is the human-meaningful identifier, the hash never leaves the DB.
- **`/health/ready` aliases**: returns `last_check_age_seconds` and `last_alert_age_seconds` (both `null` until the scheduler has run at least once). Useful for "the scheduler is stuck" alerts.
- **Audit log retention** is hard-clamped to `>=1` day inside `pruneAuditLog()` so a misconfigured `0` doesn't wipe the whole log. (Copilot review fix.)
- **Pino redaction paths** cover the obvious ones: `req.headers.authorization`, `req.headers.cookie`, `*.token`, `*.password`. Add more at the call site if a new sensitive field appears.
